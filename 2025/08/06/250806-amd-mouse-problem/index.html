关于AMD指针变白消失的解决方案

Marisa9961 发布于 2025-08-06

  • 📒 Note
  • 🏷️ AMD

鼠标指针变白消失了怎么办？

对于AMD用户来说，每当电脑出现了奇怪的显示问题，首先都要甩锅给AMD。

前阵子装完一台A卡(6750gre)的电脑后发现，每当鼠标指针指向文本框，指针就会整个变成 纯白色 ，根本找不到自己的鼠标在哪里。

虽然反复几次后指针又会恢复成黑色，但这真的很影响使用体验。

而且我曾经在用N卡的时候就从没遇见过这么奇怪的BUG，于是一口咬定这就是AMD的锅。

随后询问周围使用A卡(7700xt)的朋友们得到的回答，也都遇到过这个问题。

这下证据确凿，还能稳定复现，AMD的锅肯定是没跑了。

问题原因

随着深入搜索这个问题的深层原因，问题似乎又变得没这么简单。

在网上众说纷纭的讨论中，很多人都指向了 MPO(Multiplace Overlay) 这个看起来很陌生的名字。

经过查询后得知，这是微软早在 windows8.1 时期就引入的一种 多平面覆盖功能 。 MPO 允许图形硬件将多层内容组合到一个图像中，然后它可以在屏幕上显示。它本质上是一种硬件加速的方法，用于组合不同的内容平面，而无需涉及 CPU 或使用其他系统资源来执行软件的混合。

在许多国内外社区的讨论中，这个 MPO 早就引起过诸如黑屏卡顿以及掉驱动等常见的显卡问题，包括N卡用户也曾经深受其害。

Disabling MPO (Multiplace Overlay) can improve some desktop apps flicker or stutter issues

AMD显卡黑屏、掉驱动问题解决办法

虽然在后续的更新当中， AMD 和 Nvidia 都陆陆续续解决了一部分的适配问题，但 AMD 这边显然是还没完全修复，至少鼠标移入文本框变白的问题还没得到解决。

解决方案

网上有人会选择通过禁止浏览器硬件加速，或者修改注册表停用 MPO 来解决这些问题。

但是笔者觉得这都不是很优雅的解决方案。

禁用硬件加速后，会明显感觉到浏览器变卡顿，得不偿失。

而手动修改注册表后，又难以管理，万一将来某次驱动升级，因 MPO 的设置冲突而导致显示异常，也会让人头疼不已。

虽然直接修改鼠标样式是一个还不错的解决方法，但是笔者并不喜欢花里胡哨的指针外观，更希望以原始样式使用。

于是在上网冲浪的期间，在巴哈姆特找到了比较合适的解决方案。

使用 AMD 顯示卡將滑鼠懸停在文字或文本框上時滑鼠游標消失的解決辦法

其中分享的解决方案就是，单独替换鼠标指针在 “选择文字” 时的样式， windows 自带有多种显示样式，完全能够满足我的要求。

所以对于各位使用 windows11 的用户，只需要打开 [windows菜单]>[搜索“鼠标设置”]>[“其他鼠标设置”]>[“指针”]>[“文字选择”]>[“浏览”]>[选择“beam_r.cur”] 即可解决问题。

自带的样式与原本的样式之间的区别就是多加了一层描边，在日常使用中也不会有太大的影响，和原始默认的样式很相似。

于是这个问题就这样轻易的解决了。
